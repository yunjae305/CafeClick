# CafeClick 기존 구조 분석 및 무한 방치형 전환 포인트

## 목표

기존 CafeClick이 가진 단일 파일 클릭커 구조를 유지하면서, 엔딩 중심의 일회성 게임 루프를 무한 방치형 게임 루프로 바꾸기 위한 핵심 제약을 문서화한다.

## 기존 구조에서 확인한 제약

- 1,000,000 누적 수입에서 엔딩 오버레이가 열려 게임의 성장 구조가 사실상 종료됐다.
- 재화와 누적 수입이 일반 Number 기반이라 장기 플레이 시 안전한 정수 범위를 넘을 수 있었다.
- 자동 수입과 업그레이드 가격 계산이 커질수록 저장 포맷과 표시 포맷이 함께 불안정해질 수 있었다.
- 회차 데이터와 환생 이후에도 남아야 할 영구 데이터를 분리할 구조가 없었다.
- 상태 변경마다 렌더링과 상점 DOM을 다시 만드는 방식이라 후반부에는 화면 갱신 비용이 커질 수 있었다.

## 전환 설계 방향

- 엔딩 오버레이와 종료 조건을 제거한다.
- 내부 경제 수치는 BigInt와 고정 스케일로 관리한다.
- 현재 회차 상태는 `runState`, 영구 보상 상태는 `prestigeState`로 분리한다.
- 업그레이드 비용은 각 구매 시 갱신되는 지수형 비용 구조로 유지한다.
- 상점 버튼은 한 번만 만들고, 이후에는 텍스트와 비활성화 상태만 갱신한다.
- 게임 루프는 `requestAnimationFrame` 기준으로 돌리고, 드로우는 스로틀링한다.

## 전환 후 기대 효과

- 엔딩이 없어도 플레이 동력이 유지된다.
- 큰 숫자 연산과 저장이 안전해진다.
- 환생 시스템을 통해 휘발성 데이터와 영구 데이터의 생명주기를 분리할 수 있다.
- 후반부에도 DOM 업데이트 부하를 줄이면서 부드러운 진행이 가능하다.
