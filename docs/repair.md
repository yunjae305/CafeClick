# CafeClick 수정 내역 정리

## 수정 대상

단일 파일로 작성된 레거시 클릭커 게임의 상태 동기화 문제를 중심으로 유지보수를 진행했습니다.

## 핵심 수정

### 1. 자동 수입 동기화 버그 수정

- 자동 수입이 발생할 때 전체 UI가 아니라 일부 수치만 갱신되던 흐름을 정리했습니다.
- 자동 수입 이후에도 `refreshAll()`이 호출되도록 바꿔 마일스톤, 상점, 통계가 동시에 갱신되게 했습니다.

### 2. 렌더링 경로 단순화

- 클릭, 업그레이드 구매, 자동 수입처럼 상태를 바꾸는 모든 동작이 같은 렌더링 경로를 타도록 정리했습니다.
- 화면마다 다른 방식으로 값을 밀어 넣던 구조를 줄여 디버깅 포인트를 단순하게 만들었습니다.

### 3. 회귀 테스트 추가

- `tests/cafe-clicker.test.js`에 Node `vm` + 가짜 DOM 기반 테스트를 추가했습니다.
- 초기 렌더, 업그레이드 구매, 저장 데이터 복원, 자동 수입 반영, 엔딩 오버레이까지 검증하도록 구성했습니다.

## 검증 명령

```bash
node tests/cafe-clicker.test.js
```

## 유지보수 관점에서의 성과

- 문제 재현 경로를 문서로 남겼습니다.
- 수정 범위를 설계 문서로 먼저 고정했습니다.
- 리팩토링 결과를 테스트 코드로 다시 확인할 수 있게 만들었습니다.
