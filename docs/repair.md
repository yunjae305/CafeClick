# CafeClick 무한 방치형 전환 수정 내역

## 수정 대상

기존 엔딩형 클릭커 게임을 무한 방치형 구조로 재설계하면서 경제 시스템, 저장 포맷, UI 루프, 테스트를 함께 수정했다.

## 핵심 수정

### 1. 엔딩 제거 및 환생 구조 도입

- 1,000,000 누적 수입에서 열리던 엔딩 오버레이와 종료 조건을 제거했다.
- 대신 `황금 원두`를 지급하는 환생 버튼과 영구 수익 버프 구조를 추가했다.
- 회차 진행 데이터와 영구 보상 데이터를 분리해 상태 생명주기를 명확히 했다.

### 2. BigInt 기반 경제 시스템 전환

- 재화, 누적 수입, 판매량, 업그레이드 단계, 비용 계산을 BigInt 중심으로 바꿨다.
- 내부적으로 고정 스케일을 둬 초당 수입과 클릭 수입을 같은 방식으로 누적하게 만들었다.
- 표시용 숫자는 한국어 단위 포매팅 함수로 변환해 큰 숫자도 읽기 쉽게 표현했다.

### 3. 지수형 비용 증가와 저장 포맷 재설계

- 업그레이드 비용은 구매 때마다 비율을 곱해 다음 비용으로 갱신되게 했다.
- `localStorage` 저장 포맷을 versioned JSON으로 바꾸고, BigInt 값은 문자열 기반으로 저장하게 했다.
- 이전 구조의 저장 데이터도 가능한 범위 안에서 읽어올 수 있게 했다.

### 4. 렌더링 최적화

- 상태 변경마다 `shop` 전체를 다시 만드는 구조를 제거하고, 업그레이드 버튼을 한 번만 만든 뒤 텍스트와 비활성화 상태만 갱신하게 했다.
- `requestAnimationFrame` 루프와 드로우 스로틀링을 도입해 수입 변화가 자주 발생해도 화면 업데이트가 과도하게 일어나지 않게 했다.

### 5. 회귀 테스트 전면 교체

- 엔딩 관련 테스트를 제거했다.
- `tests/cafe-clicker.test.js`를 BigInt 포매팅, 환생 보상, 저장 복원, 지수형 비용 증가, 드로우 스로틀링 검증 중심으로 다시 작성했다.

## 검증 명령

```bash
node tests/cafe-clicker.test.js
```

## 유지보수 관점에서의 성과

- 엔딩 중심 구조를 무한 확장 가능한 시스템으로 전환했다.
- 큰 숫자 처리와 장기 상태 관리 문제를 한 번에 정리했다.
- 브라우저 렌더링과 저장 구조까지 포함한 실질적인 유지보수 범위를 문서와 테스트로 남겼다.
